# CHIMAX8.1 電梯控制系統說明書

## 【前言與系統概述】
*   **適用範圍**：本電梯控制盤適用於 **32 停靠樓以下**之電梯控制系統。
*   **電梯速度**：15 ~ 150M/分。
*   **傳輸介面**：使用專用 **4 線傳輸介面（內外 UHCB、UCDH、MCA 基板 (LAN)）**，可擴充至 32 停靠樓層，節省調試工時及線材。
*   **處理器架構**：LAN 傳輸採用多微處理機 (CPU) 系統，不佔用主程序 CPU 處理時間，反應速度快。
*   **叫車擴充板**：四線 LAN 傳輸式叫車擴充板，分為「串列式叫車傳輸板」與「並列式叫車傳輸板」，可依現場需求混合搭配使用，具備極大彈性。
*   **硬體特點**：
    *   系統線路簡單，MAX8.1 體積小、擴充功能強。
    *   可直接於主板顯示電梯狀況及故障代碼。
    *   電梯狀態參數可直接由主板設定，不需另接操作鍵盤，維修操作容易。
    *   配合傳輸式**車廂 CTBC 車廂頂控制基板之上之 I/O 介面**，活動電纜線僅需 **30 芯線** (含對講機)，簡化配線縮短工時，並達到產品規格化之目標。

# 電梯控制系統故障診斷說明書（黑版適用）

## 1. 【圖表文字解說】
本說明書定義了電梯控制系統的故障記錄格式 **`E X.YY.ZZ`**。
此格式為一個 5 碼的十六進制複合代碼，用以記錄電梯發生故障時的「歷史順序」、「變頻器與安全迴路狀態」以及「具體故障成因」：

## 3. 實務診斷案例與操作指南

### A. 故障代碼解析範例
*   **顯示內容**：`E0 AD 14`
*   **分析解譯**：
    *   **`E0`**：第 0 組（最新一次）故障紀錄。
    *   **`AD` (YY 狀態)**：
        *   高位元 `A` (10) = `8` (BR 動作中) + `2` (INV RUNNING)
        *   低位元 `D` (13) = `8` (INV MS2) + `4` (INV MS1) + `1` (INV F)
        *   *狀態結論*：故障發生時，電梯正處於正轉運行中（高速），且抱閘已釋放動作。
    *   **`14` (ZZ 故障)**：安全回路 (SAR) OFF。
    *   **整體診斷結果**：電梯在正轉高速運行中，突然發生安全回路（SAR）斷開，導致電梯緊急停車。

### B. 故障調閱與清除操作步驟
1.  **調閱最新故障**：將控制盤 (C/P) 的操作開關切換至 **`INS` (手動檢修)** 位置，按下 **`CL`** 鍵，螢幕即顯示最後一次發生的故障代碼。
2.  **翻查歷史紀錄**：在按住 **`CL`** 鍵的同時，按下 **`HUP` (上行)** 或 **`HDN` (下行)** 鍵，可依序調閱 16 次內的歷史故障紀錄。若歷史故障超過 16 次，系統會將最舊的紀錄標示為 `EF.yyzz`。
3.  **清除故障紀錄**：同時按下 **`CL` + `OP`** 鍵保持 **5 秒**，系統會清除所有歷史故障紀錄（清除後顯示 `E0.yyzz~EF.yyzz` 為空值）。
4.  **【安全注意事項】**：若要調閱故障碼，**車廂頂部的檢車（車頂檢修）操作不能切入**。若切入車頂檢修，主機板將鎖定，無法進行故障碼調閱操作。

### 3. 【系統核心操作與保護邏輯】

#### 故障調閱與清除操作步驟
1. **進入調閱模式**：必須將控制盤 (C/P) 的操作開關切換至 **`INS` (檢修)** 位置。
   * *注意：若車廂內的檢修操作開關已被切入，主機板將無法進行故障調閱。*
2. **讀取最新故障**：按下 **`CL`** 按鍵，屏幕將顯示最後一次（最新）的故障代碼。
3. **歷史紀錄翻頁**：按住 **`CL` 鍵 + `HUP` (上)** 或 **`HDN` (下)** 可依序調閱前 16 次的歷史故障紀錄。
   * *註：若故障發生超過 16 次，系統會以 `EF.yy.zz` 作為最後一次的故障紀錄存檔。*
4. **清除歷史紀錄**：同時按住 **`CL` + `OP`** 鍵保持 **5 秒**，即可清除主板內的所有故障代碼紀錄（`E0.yy.zz` ~ `EF.yy.zz`）。

#### 異常狀態下的系統強制動作
* **當發生門短路故障 (ZZ=16, 17, 18)**：
  電梯將強制保持在**開門狀態**，且 **`AL` (警鈴) 保持響起**。
* **當發生抱閘電氣黏著 (ZZ=06)**：
  安全保護機制啟動，使 **`INVP` 跳脫**，切斷 `B+` 與 `B-` 之間的煞車電源。
* **當發生抱閘機械故障 (ZZ=B6) 或 溜車偵測 (ZZ=E6)**：
  1. 在開門狀態下，系統自動**取消所有叫車信號**。
  2. 觸發 **`AL` 響鈴**、播報語音並顯示警示通知。
  3. 變頻器強制保持 **`0 速度` 輸出**以鎖定車廂防止滑移，直到車門完全關妥。
  4. 車門關妥後，電梯會以安全模式**慢速回歸至頂樓**，抵達後**不開門**，隨後釋放 0 速度控制，停機鎖梯。

# 電梯控制系統參數調整說明書（黑版適用）

## 一、 控制板按鍵功能說明
在參數調整模式下，主控板上的按鍵定義如下：
*   **CL**：「$\rightarrow$」欄位切換鍵。
*   **OP**：「SET」資料設定儲存鍵。
*   **HUP**：「+」設定欄位內容值 +1。
*   **HDN**：「-」設定欄位內容值 -1。
*   *備註：系統所有參數數值皆採用 **16 進制 (Hexadecimal)** 數值表示。*

## 二、 參數調整操作步驟
1.  **安全準備**：請先將安全迴路切斷（**BXC 短路**），並將電梯控制板切換於**手動**狀態。
2.  **進入模式**：切斷電梯電源開關。重新送電之時，**按住 MODE 按鍵不放持續 2 秒鐘**，系統即進入參數調整模式。
3.  **數值修改**：
    *   顯示器將顯示參數代碼（如 A0-00）及其設定值。
    *   按 **$\rightarrow$ 鍵**可切換至欲設定的數值欄位（此時該欄位會閃爍）。
    *   按 **+ 或 - 鍵**改變數值至目標參數值。
    *   修改完成後，再按一次 **$\rightarrow$ 鍵**進入下一欄位（閃爍位置改變）。
4.  **資料儲存**：更改資料欄位內的數值後，**必須按下 OP (SET) 鍵**，新數值才會成功儲存。
5.  **退出模式**：參數設定完成後，將安全迴路及手自動迴路回復正常狀態。**將電梯重新送電**，系統即可脫離參數設定模式並投入正常運行。

# 電梯控制系統說明書：7.2 電梯運轉參數調整（藍綠版適用）

## 一、 按鍵操作說明
在參數設定模式下，主機板按鍵功能定義如下：
*   **CL**：對應「 `→` 」鍵，用於**欄位切換**。
*   **OP**：對應「 `SET` 」鍵，用於**資料設定與儲存**。
*   **HUP**：對應「 `+` 」鍵，用於**設定欄位內容值 +1**。
*   **HDN**：對應「 `-` 」鍵，用於**設定欄位內容值 -1**。
*   **注意**：此系統中所有參數數值皆為 **16 進制（Hexadecimal）**。

---

## 二、 參數調整步驟方法
1.  **安全準備**：請將安全迴路切斷，並將控制板切換至**手動（MANUAL）**狀態。
2.  **進入模式**：切斷電梯主電源。在重新送電的瞬間，**按住 MODE 按鍵不放持續 2 秒**，系統即會進入參數調整模式。
3.  **密碼驗證**：
    *   主機板顯示器將顯示 `AA.0000`。
    *   請輸入正確的進入密碼後按下 **OP (ENT)** 鍵。密碼正確始可進入參數設定。
    *   *註：若系統無設定密碼（即 A6-01, A6-00 設為 `0000` 時），則會直接進入參數設定畫面 `A0.00.XX`。*
4.  **切換與修改**：
    *   顯示器會顯示當前參數代碼與其設定值。
    *   按下 **CL (→)** 鍵可切換至欲修改的設定欄位（此時該欄位會閃爍）。
    *   按下 **HUP (+)** 或 **HDN (-)** 鍵改變數值至目標設定值。
    *   修改完成後，再按一次 **CL (→)** 鍵可進入下一個欄位（閃爍位置會改變）。
5.  **儲存設定**：修改資料欄位內的數值後，**必須按下 OP (ENT) 鍵**才能成功儲存數值。
6.  **回復運轉**：參數設定完成後，請將安全迴路及手自動迴路回復正常狀態，並**重新送電**，系統即會脫離參數設定模式並以新參數運行。

你好！我是電梯控制系統專家。這張圖紙是針對 **15 ~ 150 M/M（米/分鐘）中高速度電梯** 的變頻器控制端子接線圖，並以 **HITAKE 1200 變頻器** 作為實際接線範例。

為了讓 AI 知識庫（RAG 系統）能夠進行精準的語意檢索與邏輯推理，我已將此圖紙的所有核心線路、邏輯、參數與接線定義轉換為結構化的繁體中文文本。

### 1. 【圖表文字解說】

本圖紙為**「15 ~ 150M/M 電梯變頻器控制端子接線圖」**，主要闡述電梯主控板與變頻器（以 HITAKE 1200 為主）之間的控制信號對接標準、速度控制編碼邏輯，以及編碼器（Encoder）回授訊號的接線規範。

其核心電路與運作邏輯包含以下四大重點：
1. **主迴路安全互鎖（IMS）**：若電梯未裝設變頻器與馬達之間的「保護電磁接觸器」，必須將主控板 `CN14-5`（IML動作輸出）與 `CN14-13`（IML動作檢出）兩端子進行強制短接，以避免系統判定故障。
2. **多段速運轉控制**：系統透過三組數位輸入端子（`MS1`、`MS2`、`MS3/JOG`）進行二進制編碼組合，共可定義 8 種不同的電梯運行速度（包含著床速、檢修手動速、救出速及三段不同的全速段）。
3. **變頻器運作回授**：利用 `Y1`（運行信號）與 `Y2`（速度到達訊號）等繼電器輸出，將變頻器狀態回授給主控板，確保運轉安全。
4. **PG 速度回授卡（PGB_2K）接線**：針對「推挽式輸出（Push-Pull）」與「開集極輸出（Open Collector）」兩種不同類型的編碼器，提供明確的端子排（TA1、TA2）配線與線色定義（白、黑、紅、綠、遮蔽網），確保閉迴路控制（Closed-loop Control）的精準度。
